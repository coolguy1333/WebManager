Host a site at the domain root

This serves one site directly at https://{{ domain.name }}. A wildcard entry does not cover this address.

  1. Open Cloudflare Zero Trust -> Networks -> Tunnels -> your tunnel -> Public Hostnames.
  2. Add hostname {{ domain.name }} with no subdomain or path.
  3. Set Service type to HTTP and URL to localhost:8080. If cloudflared runs elsewhere, use WEBMANAGER-LAN-IP:8080.
  4. Save it and confirm Cloudflare creates a proxied DNS route for the zone apex.
  5. In WebManager, choose Domain root while deploying exactly one site folder.
  6. Test locally with curl -I -H 'Host: {{ domain.name }}' http://127.0.0.1:8080/.
HTTPS must cover {{ domain.name }}. Cloudflare normally includes the zone apex when the domain is active and proxied.
{% endif %}

Host sites on subdomains

This serves addresses such as project.{{ domain.name }} and supports multiple sites.

  1. Add a second Tunnel public hostname *.{{ domain.name }}.
  2. Use the same service, http://localhost:8080, or http://WEBMANAGER-LAN-IP:8080 for a remote connector.
  3. Confirm the proxied wildcard DNS record exists.
  4. Test DNS with nslookup test.{{ domain.name }} 1.1.1.1.
  5. Test routing with curl -I -H 'Host: test.{{ domain.name }}' http://127.0.0.1:8080/.
Always include :8080. A target such as http://192.168.10.20 uses port 80 and may reach Debian's default Nginx site instead of WebManager.
